Persistent Intelligence

The 4-Layer Memory System

Memory is what transforms a tool into a partner. JARVIS remembers everything — and knows what to use when.

⚡ Working Memory

Last 20 conversational turns, active session state, current task context. Held in-process. Fast retrieval. Wiped on session close.

  • Current conversation buffer
  • Active task context
  • Pending confirmations / follow-ups

📖 Episodic Memory

Stored in Supabase pgvector + Pinecone. Summarized events, decisions made, outcomes observed. Semantically searchable.

  • Every significant decision + outcome
  • Business wins and losses
  • Important conversations summarized
  • Daily digest snapshots

🧬 Semantic Memory

Structured facts in Supabase tables. Your preferences, goals, people you know, recurring patterns. Updated continuously.

  • Your 1/3/10-year goals
  • People & relationship notes
  • Preferences + communication style
  • Known constraints (immigration, tax, etc.)

📜 Procedural Memory

Your SOPs, frameworks, decision rules, and personal playbooks. Loaded into relevant agent contexts when needed.

  • Proposal writing playbook
  • Client onboarding SOP
  • Weekly review template
  • How you handle X decision type

Security & Privacy

Non-Negotiable Security Design

You're giving this system access to your entire life. Security isn't optional.

🔐

Secrets Management

All API keys in environment variables or HashiCorp Vault. Never in code or logs. Rotate per-integration. Separate dev/prod credentials.

🏰

Auth Boundaries

Each agent runs with least-privilege access. Scoped tokens per integration. No agent can access another's credentials. Auth at every API endpoint.

📝

Audit Logging

Every agent action is logged with timestamp, agent ID, input summary, and output. Tamper-proof. Stored for 90 days. Reviewable in Mission Control.

🛡

Encrypted at Rest

All personal data encrypted in Supabase. Vector embeddings anonymized where possible. Sensitive fields (financial data, health) double-encrypted.

⛔

Confirmation Gates

Any irreversible action (send email, delete, publish, pay) requires explicit confirmation via Telegram before execution. No silent mutations.

💸

Cost Guardrails

Per-agent token budgets. Daily API spend caps with alerts. Model routing logic to minimize cost (Haiku first, escalate only when needed).
